# Hiring Plan: Founding Engineer

## Role Definition

**Title:** Founding Engineer
**Type:** Full-time first hire (agent)
**Reports to:** CEO

## What the Company Does

A trading alpha/opportunities dashboard that aggregates crypto analysis, prediction markets, social signals, and on-chain data into simple, high-conviction decision indicators for retail users. Not a trading platform—a decision-support dashboard with AI agents.

## What the Founding Engineer Will Build

- Web dashboard aggregating crypto/opportunity signals
- Data pipelines from exchange APIs, news feeds, social sentiment, prediction markets
- AI agents that distill complex analysis into simple indicators
- User-facing UI that presents only the highest-conviction opportunities

## Technical Stack (Initial)

- **Frontend:** React/TypeScript, modern UI
- **Backend:** Node.js or Python, REST + WebSocket APIs
- **Data:** Exchange APIs (Binance, Coinbase), news/sentiment APIs, prediction markets (Polymarket, Kalshi)
- **Infrastructure:** Cloud-hosted (AWS/GCP), containerized

## Skills Needed

- Full-stack development (React + Node/Python backend)
- Experience with financial data APIs or willingness to learn
- Strong sense for UX—distilling complexity into simplicity
- Self-directed; comfortable with ambiguity

## Hiring Process

1. **Sourcing:** OpenClaw invite to recruit agent candidates
2. **Technical Screen:** Assess full-stack capability and financial data experience
3. **Culture/Alignment Interview:** Confirm shared vision for the product
4. **Offer:** Hire via Paperclip/OpenClaw onboarding

## Roadmap (Phased)

### Phase 1 — MVP (Weeks 1-4)
- Data pipeline: crypto prices + basic indicators from exchange APIs
- Simple React dashboard showing top opportunities
- Core AI signal agent: summarize market sentiment
- Basic authentication / user preferences

### Phase 2 — Signal Expansion (Weeks 5-8)
- Social sentiment ingestion (Twitter/X, Reddit)
- Prediction market integration (Polymarket, Kalshi)
- Enhanced AI agent with multi-source synthesis
- Alerting / notification system

### Phase 3 — Polish & Growth (Weeks 9-12)
- User onboarding flow
- Portfolio tracking integration
- Mobile-friendly UI
- Community features (shared watchlists, signal discussions)

## Ownership Model

- CEO: strategy, vision, fundraising, hiring
- Founding Engineer: all technical execution, architecture, data
- Work delegation via Paperclip tasks starting immediately

## Success Metrics

- MVP live and usable by end of Phase 1
- Top 3 crypto opportunities surfaced daily
- User activation and retention in Phase 3
